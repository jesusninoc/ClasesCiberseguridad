### 🔹 Ejemplo de comandos
```bash
sudo apt update
sudo apt install -y apktool openjdk-17-jdk

apktool d -f -r app-debug.apk -o original_nres
nano original/res/values/strings.xml
apktool b original_nres -o app-mod-unsigned.apk
keytool -genkeypair -v -keystore lab.keystore -alias labkey -keyalg RSA -keysize 2048 -validity 3650
jarsigner -verbose -keystore lab.keystore app-mod-unsigned.apk labkey
```

> [!IMPORTANT]
> En contexto docente conviene presentar esta práctica como **análisis de integridad, modificación controlada de recursos y firma de aplicaciones en laboratorio**.

# ✅ Criterio de uso docente

## Enfoque recomendado
Estas prácticas pueden emplearse para:
- análisis de configuraciones inseguras,
- estudios de caso,
- laboratorios de bastionado,
- ejercicios forenses,
- detección de artefactos sospechosos,
- revisión de metadatos, trazas y evidencias,
- auditorías básicas en sistemas propios o autorizados,
- análisis de aplicaciones web vulnerables en laboratorio.

## Enfoque que conviene evitar
No deberían plantearse como:
- guía para comprometer sistemas de terceros,
- intrusión sin autorización,
- despliegue real de artefactos maliciosos,
- evasión de controles en sistemas ajenos,
- acceso no consentido a redes, dispositivos o servicios.
